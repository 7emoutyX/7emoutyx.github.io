Introduction

During my two-month internship in the Maintenance Department at Fujikura Automotive Kenitra.I implemented a real-time conveyor production monitoring system using a Raspberry Pi to enhance production tracking. This system was developed as a solution to replace the old manual method of production monitoring, providing a more efficient and accurate approach. In this Blog I will share with u the full project SW and HW , including source code , schematics , pcb layout …

SOFTWARE ASPECT :

The project was fully built in C, and as part of this, I went beyond the standard implementation by patching the Raspbian kernel to achieve real-time operating system (RTOS) capabilities, optimizing the system’s performance for critical tasks. Preparing the modified Raspbian Kernel :

Adding PREEMPT-RT Patch to Raspbian Kernel

This guide provides the steps to patch your Raspbian kernel with PREEMPT-RT for enabling real-time capabilities on Raspberry Pi.

1. Prepare Your Raspberry Pi

Ensure your Raspberry Pi is up-to-date:

sudo apt update && sudo apt upgrade

Install essential packages for building the kernel:

sudo apt install bc bison flex libssl-dev make

2. Download the Kernel Source

Navigate to the directory where you’ll store the kernel source code:

cd /usr/src

Download the Raspbian kernel source code:

git clone --depth=1 https://github.com/raspberrypi/linux
cd linux

3. Download the PREEMPT-RT Patch

Download the latest PREEMPT-RT patch that matches your kernel version from the official repository: PREEMPT-RT Patches For example, for kernel 5.10, you can find patches like patch-5.10.xx-rt.patch.gz. Download and extract the patch:

wget https://cdn.kernel.org/pub/linux/kernel/projects/rt/5.10/patch-5.10.xx-rt.patch.gz
gunzip patch-5.10.xx-rt.patch.gz

4. Apply the Patch

Apply the PREEMPT-RT patch to your downloaded kernel source:

patch -p1 < patch-5.10.xx-rt.patch

Verify the patch was applied successfully without errors.

5. Configure the Kernel

Load the default kernel configuration for your Raspberry Pi:

KERNEL=kernel7
make bcm2709_defconfig

Enable the PREEMPT-RT configuration:

make menuconfig

Go to “Kernel Features” → Enable “Preemptible Kernel (Basic RT)” or “Fully Preemptible Kernel (RT)”.

Exit the menu and save the configuration.

6. Compile the Kernel

Compile the kernel with real-time patches:

make -j$(nproc) zImage modules dtbs

7. Install the New Kernel

After compilation, install the kernel modules:

sudo make modules_install

Install the kernel, device tree blobs (dtbs), and all of theese files :

sudo cp arch/arm/boot/zImage /boot/kernel7.img
sudo cp arch/arm/boot/dts/*.dtb /boot/
sudo cp arch/arm/boot/dts/overlays/*.dtb* /boot/overlays/
sudo cp arch/arm/boot/dts/overlays/README /boot/overlays/

8. Update Bootloader (u can skipp this step)

update the bootloader configuration (/boot/config.txt) to use the new kernel.

9. Reboot and Verify

Reboot your Raspberry Pi to apply the changes:

sudo reboot

After rebooting, check that the PREEMPT-RT patch is active by running:

uname -a

You should see a kernel version with “PREEMPT RT” in the name, confirming that the real-time patches are active.

10. Test and Optimize

u can test the real-time performance of these patches by running tools like cyclictest:

sudo apt install rt-tests
cyclictest --smp --priority=80 --interval=200 --distance=0 -m

Hardware ASPECT :

During my internship with the maintenance team, I encountered an interesting situation: one of the cable joining machines went down. Upon inspecting its PCB, I discovered something unexpected. Typically, I’ve seen industrial boards using integrated compute units, often powered by ARM or x86-based CPUs. Some even utilize FPGA or RISC-V architectures. However, this time, I came across a board powered by a Raspberry Pi Compute Module 1.

At first, it struck me as odd. Why use a Raspberry Pi Compute Module in an industrial setting? But after giving it some thought, I realized the brilliance behind this approach. Leveraging a Raspberry Pi Compute Module offers a number of benefits:

  • Modularity: It allows for easy upgrades and maintenance.
  • Open-source ecosystem: You gain access to a vast array of open-source software and hardware.
  • Cost efficiency: Raspberry Pi solutions are generally more affordable compared to traditional industrial compute units.

Of course, there are some trade-offs, such as potential limitations in performance or durability compared to purpose-built industrial CPUs, but overall, I found the design clever and innovative. This experience inspired me to take on a project of my own. Rather than building a Raspberry Pi HAT (which was my original plan), I decided to go a step further and design a full-fledged PCB based on the Raspberry Pi Compute Module 4.

I designed a 4-layer PCB using Altium, incorporating high-speed signal traces for HDMI, Ethernet, and USB, along with an integrated power module to ensure the board could handle the demands of an industrial environment.

It’s been an exciting journey re-engineering this board, and I’m thrilled to see how versatile the Raspberry Pi can be, even in industrial applications. This experience has deepened my appreciation for the power of open-source hardware and inspired me to push the boundaries in my own work.
